# NextJS Skill

## When to use

Use this skill for Next.js pages, app router segments, server components, route handlers, layouts, metadata, and SSR behavior.

## Rules

- Follow the project's router style before adding pages or app-directory structure.
- Keep server-only code out of client components and mark client components explicitly.
- Preserve data-fetching, caching, and revalidation behavior when editing routes.
- Keep shared UI and route helpers in existing project locations.
- Test rendered routes, route handlers, and build/type checks where available.

## Verification

Run the focused Next.js lint, typecheck, build, or route test command when available.
