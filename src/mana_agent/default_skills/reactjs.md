# ReactJS Skill

## When to use

Use this skill for React components, hooks, JSX/TSX state, props, effects, context, and component-level tests.

## Rules

- Reuse existing component, hook, and state-management patterns.
- Keep effects dependency-safe and avoid derived state when props or memoization are enough.
- Preserve accessibility semantics for interactive elements.
- Keep component boundaries small and colocate tests with the existing convention.
- Verify both loading/error and normal interaction states for user-facing changes.

## Verification

Run focused component tests, lint, typecheck, or build commands that cover the changed React files.
